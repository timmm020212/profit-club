# ИДЕЯ ПРОЕКТА — Profit Club

> Документ для AI-агентов. Написан по методологии Spec-First.

---

## 1. Проблема

Администратор салона красоты тратит 2–3 часа в день на ручное управление:
- Записывает клиентов в мессенджеры/по телефону
- Согласует расписание с мастерами через WhatsApp
- Не видит аналитику: кто ходит, что популярно, сколько зарабатывает каждый мастер
- Мастера не получают уведомления о новых записях своевременно
- Клиенты забывают о записях → потерянные слоты

## 2. Решение

Profit Club — CRM-система для одного салона красоты (г. Ставрополь).

**Пошаговый процесс работы:**
1. Администратор создаёт услуги и мастеров в веб-интерфейсе
2. Администратор назначает рабочие слоты мастерам
3. Мастер подтверждает слоты через Telegram-бот (кнопки ✅/❌)
4. Клиент выбирает услугу и мастера на сайте, видит свободное время
5. Клиент записывается через форму на сайте
6. За 1 час до записи клиент получает напоминание в Telegram
7. Администратор видит всё расписание на дашборде с тайм-линией

## 3. Целевая аудитория

**Основная — администратор (1 человек):**
- Управляет всеми записями, мастерами, услугами
- Работает через веб-интерфейс `/admin`
- Получает уведомления через Telegram-бот (admin-бот)

**Вторичная — мастера (3–10 человек):**
- Подтверждают/отклоняют рабочие слоты через Telegram
- Могут запрашивать изменение времени
- Работают только через бот (masters-бот)

**Третичная — клиенты (10–200 человек):**
- Просматривают услуги на сайте
- Записываются онлайн
- Получают напоминания через клиентский Telegram-бот

## 4. Архитектура системы

```
[Клиент] → [Next.js сайт] → [API Routes] → [SQLite/PostgreSQL]
[Клиент] → [Telegram бот (клиентский)]      ↑
[Мастер] → [Telegram бот (мастеров)]   → [DB]
[Админ]  → [Next.js /admin + Telegram] → [DB]
```

**Текущий стек:**
- Frontend: Next.js 15 App Router + TypeScript + Tailwind CSS
- Backend: Next.js API Routes (serverless)
- БД: SQLite (better-sqlite3 + Drizzle ORM) → планируется PostgreSQL
- Auth: NextAuth v4 (admin) + пароль+Telegram (клиенты)
- Боты: Telegraf (3 бота: admin, masters, client)
- Деплой: Dockhost (планируется)

## 6. Монетизация

**Текущая фаза:** Инструмент для одного салона, не SaaS.
**Нет планов по монетизации** на данном этапе.

## 7. Конкуренты

| Конкурент | Что делает | Чего не хватает |
|-----------|-----------|-----------------|
| Yclients | Полная CRM, онлайн-запись | Дорого, сложно, нет Telegram-интеграции |
| 1С: Салон | Учёт услуг | Устаревший UI, нет онлайн-записи |
| Telegram-боты (самописные) | Уведомления | Нет веб-интерфейса для клиента |
| Google Forms | Запись | Не отображает свободное время мастера |

**Преимущество Profit Club:** Telegram-first подход + современный веб-интерфейс + единая база данных

## 8. MVP (сделано) vs v2 (планируется)

**MVP — текущее состояние:**
- Управление услугами, мастерами, слотами
- Онлайн-запись клиентов
- 3 Telegram-бота
- Напоминания

**v2 — приоритеты:**
1. Аналитика для владельца (доходы, популярные услуги, загруженность мастеров)
2. Личный кабинет клиента на сайте (история записей, редактирование профиля)
3. Запись через Telegram-бот (не только сайт)
4. Миграция на PostgreSQL (масштабируемость + Dockhost)

## 9. Риски

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| SQLite не выдержит нагрузку | Средняя | Запланирована миграция на PostgreSQL |
| Бот 409 Conflict (несколько инстансов) | Высокая | Один инстанс на Dockhost через Docker |
| Потеря данных (SQLite файл) | Средняя | Бэкап + переход на PostgreSQL |
| Сложность деплоя 3-х ботов | Средняя | Docker Compose с 4 сервисами |

## 10. Технические детали

**Переменные окружения (.env.local):**
- `TELEGRAM_BOT_TOKEN` — клиентский бот
- `MASTERS_BOT_TOKEN` — бот мастеров
- `NEXTAUTH_URL` — URL сайта
- `NEXTAUTH_SECRET` — секрет NextAuth

**Ключевые бизнес-правила:**
- Даты ≤1 день: только подтверждённые слоты мастера
- Даты 2+ дней: сначала слот мастера, иначе часы салона 08:00–20:00
- Роль мастера матчится с категорией услуги (substring match)
- Дата всегда через `getFullYear/getMonth/getDate` (не `toISOString`)
