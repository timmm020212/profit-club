## Текущая фаза: Стабилизация и деплой

**Цель:** довести MVP до стабильного состояния и задеплоить на Dockhost.

### Приоритет 1 — Миграция на PostgreSQL

**Почему важно:**
- SQLite не подходит для Docker-деплоя (проблемы с concurrent writes)
- PostgreSQL — единственный вариант для Dockhost с несколькими процессами (Next.js + 3 бота)
- `db/index-postgres.ts` уже готов, нужно только переключить

**Шаги миграции:**

```
1. Поднять PostgreSQL в docker-compose.yml
   services:
     db:
       image: postgres:16-alpine
       environment:
         POSTGRES_DB: profit_club
         POSTGRES_USER: postgres
         POSTGRES_PASSWORD: ${DB_PASSWORD}

2. Обновить .env.local:
   DATABASE_URL=postgresql://postgres:PASSWORD@localhost:5432/profit_club

3. Изменить drizzle.config.ts → указать на drizzle.config.postgres.ts

4. Обновить db/index.ts:
   export * from './index-postgres'  (вместо index-sqlite)

5. Выполнить db:push на новую PostgreSQL
   npm run db:push

6. Перенести данные из SQLite → PostgreSQL (один раз, вручную или скриптом)

7. Убедиться, что все боты импортируют из ../db (не ../db/index-sqlite)
```

**ВАЖНО:** После миграции удалить `better-sqlite3` из зависимостей, `profit_club.db` в .gitignore.

### Приоритет 2 — Docker-деплой на Dockhost

**docker-compose.yml структура:**
```yaml
services:
  db:          # PostgreSQL
  web:         # Next.js (npm run build && npm start)
  bot-client:  # tsx telegram-bot/client-simple.ts
  bot-masters: # tsx telegram-bot/masters-bot-full.ts
  bot-admin:   # tsx telegram-bot/admin-bot-full.ts
```

**Правило:** один инстанс каждого бота → нет 409 Conflict ошибок.

**Страница:** `/admin/analytics`

Что показывать:

### Блок Б: Личный кабинет клиента

**Страница:** `/profile` (защищена сессией клиента)

Что показывать:
- История записей (прошедшие + предстоящие)
- Возможность отменить предстоящую запись
- Редактирование имени, телефона
- Статус Telegram-привязки

**Новые API:**
- `GET /api/clients/me` — данные текущего клиента (по сессии или Telegram ID из localStorage)
- `PATCH /api/clients/me` — обновить профиль
- `DELETE /api/appointments/[id]/cancel` — отменить запись (свою)

**Авторизация клиента:** хранить в localStorage `profit_club_telegram_id` и `profit_club_user_name`.
При открытии `/profile` — проверять через `GET /api/clients/me?telegramId=XXX`.

### Блок В: Запись через Telegram-бот

**Сценарий:**
1. Клиент нажимает "📅 Записаться" в боте
2. Бот показывает список категорий услуг (inline кнопки)
3. Клиент выбирает категорию → список услуг
4. Клиент выбирает услугу → список доступных мастеров
5. Клиент выбирает мастера → inline календарь дат
6. Клиент выбирает дату → список свободных слотов времени
7. Клиент подтверждает → `POST /api/appointments`

**Сложность:** многошаговый диалог через `bot.action()` + хранение контекста в `ctx.session` (нужен telegraf-session или Redis).

**Предусловие:** Блок А и Б важнее. Начинать только после их реализации.
