# Передача контекста следующему агенту

Когда пользователь просит сохранить контекст или ты чувствуешь что контекст скоро закончится — создай файл `HANDOFF.md` в корне проекта.

## Структура файла HANDOFF.md

```markdown
# Передача контекста — [дата]

## Что было сделано в этой сессии
- [Конкретные выполненные задачи с указанием файлов]

## Текущее состояние проекта
- [Что работает, что сломано, что в процессе]
- [Какие серверы запущены, на каких портах]
- [Какие ветки git активны]

## Незавершённые задачи
- [ ] [Задача 1 — что осталось сделать]
- [ ] [Задача 2 — что осталось сделать]

## Известные проблемы
- [Баги, ошибки, workaround-ы которые нужно помнить]

## Важный контекст
- [Решения которые были приняты и почему]
- [Что пользователь хочет сделать дальше]
- [Предпочтения пользователя которые выяснились в процессе]

## Следующие шаги (по приоритету)
1. [Первое что нужно сделать]
2. [Второе]
3. [Третье]
```

## Правила

1. **Будь конкретен** — указывай точные файлы, строки, команды. Не "поправили импорты", а "заменили `@/db/schema-sqlite` на `@/db/schema` в 25 файлах в `app/`"
2. **Включай контекст решений** — почему выбрали этот подход, а не другой
3. **Указывай состояние окружения** — Docker, БД, порты, запущенные процессы
4. **Пиши для агента** — не для человека. Агент должен прочитать файл и сразу продолжить работу без дополнительных вопросов
5. **Обнови memory** — если есть важная информация для будущих сессий, сохрани её в memory-систему тоже

## Как использовать

Пользователь может сказать:
- "сохрани контекст" / "передай контекст" / "запиши что мы сделали"
- "скоро закончим, сохрани прогресс"
- "/handoff"

После создания `HANDOFF.md` скажи пользователю:
> Контекст сохранён в `HANDOFF.md`. В начале следующей сессии скажи новому агенту: "прочитай HANDOFF.md и продолжи работу".
