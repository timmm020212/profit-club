# Service Duration Fix & Break Gaps

## Overview

Три связанных исправления: корректный расчёт endTime записи по реальной длительности услуги, визуальные слоты "Перерыв" в админке, уведомления мастеру о перерывах.

## Problem

1. Услуга 80 мин записывается как 14:00–15:00 (теряется реальная длительность)
2. Зазоры между записями (например 15:20–15:30) никак не отображаются в админке
3. Мастер не знает о коротких перерывах в расписании

## Fix 1: Duration в endTime

При создании записи endTime должен рассчитываться как `startTime + service.duration` (в минутах).

**Затронутые места:**
- `app/api/appointments/route.ts` POST — проверить что endTime считается из duration
- `telegram-bot/client/booking-flow.ts` — уже корректно (slot.end = startTime + duration)
- Слоты генерируются с шагом 30 мин, но endTime записи не привязан к сетке (80 мин = 14:00–15:20)

**Логика available-slots не меняется** — она уже проверяет пересечения по реальным startTime/endTime записей. Слот 15:00–15:20 будет заблокирован overlap-проверкой, следующий свободный — 15:30.

## Fix 2: Визуальные перерывы в админке

**Где:** компонент расписания мастера в админке (отображение дневного графика).

**Логика (на клиенте, при рендере):**
1. Получить все confirmed записи мастера на дату, отсортированные по startTime
2. Для каждой пары соседних записей: если `endTime[i] < startTime[i+1]` — зазор
3. Если зазор > 0 и < 30 мин — вставить визуальный блок "Перерыв"
4. Стиль: пунктирная рамка (`border-dashed`), серый цвет, текст "☕ Перерыв" + время (напр. "15:20–15:30, 10 мин")

**В БД ничего не хранится** — вычисляется на лету.

## Fix 3: Уведомление мастеру о перерыве

**Когда:** после создания записи (и через бота, и через сайт API).

**Логика:**
1. После INSERT appointment — загрузить все confirmed записи мастера на эту дату
2. Найти зазоры между соседними записями (сортировка по startTime)
3. Если найден зазор > 0 мин и < 30 мин (минимальный слот не влезет) — отправить уведомление

**Сообщение мастеру:**
```
☕ Перерыв 10 мин

📅 25 марта (вт)
🕐 15:20–15:30
```

Отправляется через `MASTERS_BOT_TOKEN` (HTTP API), тем же способом что `notifyMasterNewAppointment`.

**Функция:** `notifyMasterBreak` в `telegram-bot/client/notify-master.ts` (для бота) и дублирование логики в `app/api/appointments/route.ts` (для сайта).

## Что НЕ меняется

- Шаг сетки слотов (30 мин)
- UI клиента — видит только свободные слоты, перерывы не показываются
- Существующие записи в БД — не пересчитываются
