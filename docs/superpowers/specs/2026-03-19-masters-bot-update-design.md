# Masters Bot Update

## Overview

Обновление мастерского Telegram-бота: единый раздел "Расписание" с полной картиной дня и навигацией, рабочий функционал изменения рабочего дня, настройки уведомлений.

## Главное меню

```
📅 Расписание
🔄 Изменить рабочий день
⚙️ Настройки
```

Убраны: "Записи на сегодня" (объединено с расписанием).

## 1. Расписание

Показывает полное расписание на ближайший рабочий день (confirmed workSlot). Если сегодня рабочий — сегодня, иначе — ближайший.

### Формат сообщения

```
📅 Расписание — пт, 20 марта

🕐 09:00–18:00 (смена)

12:00–13:20 💇 Стрижка женская — Мария К.
☕ 13:20–13:30 перерыв (10 мин)
13:30–14:50 💇 Окрашивание — Ольга П.
☕ 14:50–15:00 перерыв (10 мин)
15:00–16:20 💇 Укладка — Екатерина В.
🕐 16:20–18:00 свободно

[← Пред. день] [След. день →]
```

### Логика

- Записи: время + услуга + имя клиента (из `appointments` JOIN `services`)
- Перерывы (зазор > 0 и < 30 мин между соседними записями): "☕ HH:MM–HH:MM перерыв (N мин)"
- Свободные окна (зазор ≥ 30 мин): "🕐 HH:MM–HH:MM свободно"
- Начало дня до первой записи: свободно (если ≥ 30 мин)
- Конец дня после последней записи: свободно (если ≥ minServiceDuration) или "🏁 свободны с HH:MM"
- Если нет записей: "Нет записей, весь день свободен"

### Навигация

Inline кнопки `[← Пред. день]` `[След. день →]`. Переключают между днями с confirmed workSlots. Если предыдущего/следующего нет — кнопка не показывается.

Callback data: `schedule_prev_YYYY-MM-DD` / `schedule_next_YYYY-MM-DD` (текущая дата в callback для определения направления).

## 2. Изменить рабочий день

### Шаг 1: выбор дня

Показать список ближайших рабочих дней (confirmed workSlots, workDate >= сегодня) как inline кнопки:

```
📅 Выберите рабочий день:

[20.03 пт, 09:00–18:00]
[22.03 вс, 10:00–18:00]
[← Назад]
```

### Шаг 2: действие

```
Выбран: 20.03 пт, 09:00–18:00

[🕐 Изменить время]
[❌ Отменить рабочий день]
[← Назад]
```

### Изменить время

Бот спрашивает новое время текстом: "Введите новое время в формате ЧЧ:ММ-ЧЧ:ММ"
Валидация формата. Создаёт `workSlotChangeRequest` с `type: "time_change"`, `suggestedStartTime`, `suggestedEndTime`, `status: "pending"`.
Ответ: "Запрос на изменение времени отправлен администратору ✅"

### Отменить день

Подтверждение: "Вы уверены? Запись на отмену отправится администратору" → `[Да, отменить]` `[Нет]`
Создаёт `workSlotChangeRequest` с `type: "cancel"`, `status: "pending"`.
Ответ: "Запрос на отмену рабочего дня отправлен администратору ✅"

## 3. Настройки

### Формат

```
⚙️ Настройки уведомлений

✅ Новые записи
✅ Отмены записей
✅ Перерывы
❌ Утреннее напоминание

Нажмите чтобы переключить:
```

Каждая строка — inline кнопка-toggle. Нажатие переключает ✅↔❌ и обновляет сообщение.

### Хранение

Новое поле `notificationSettings` (TEXT, JSON) в таблице `masters`. Формат:

```json
{
  "newAppointments": true,
  "cancellations": true,
  "breaks": true,
  "morningReminder": false
}
```

По умолчанию (null или пустое) — всё включено кроме `morningReminder`.

### Применение настроек

Перед отправкой уведомления мастеру — проверять соответствующий флаг:
- `notifyMasterNewAppointment` → проверяет `newAppointments`
- `notifyMasterCancellation` → проверяет `cancellations`
- `notifyMasterBreak` / `notifyMasterEarlyFinish` → проверяет `breaks`
- Утреннее напоминание (будущая фича) → `morningReminder`

## Изменения в БД

Одно новое поле: `ALTER TABLE masters ADD COLUMN notificationSettings TEXT`
