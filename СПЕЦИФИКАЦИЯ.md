## API эндпойнты

### Услуги
| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/services | Список всех активных услуг с category |
| POST | /api/services | Создать услугу (admin) |
| PUT | /api/services/[id] | Обновить услугу (admin) |
| DELETE | /api/services/[id] | Удалить услугу (admin) |
| PUT | /api/services/order | Изменить порядок (admin) |

### Записи
| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/available-slots | Свободные слоты. Params: masterId, serviceId, date |
| POST | /api/appointments | Создать запись |
| GET | /api/appointments | Список записей (admin) |
| PATCH | /api/appointments/[id] | Изменить статус |

**GET /api/available-slots** — ключевая логика:
- Дата ≤1 день: только `workSlots.isConfirmed = true`
- Дата ≥2 дня: сначала confirmed workSlot, иначе 08:00–20:00
- Роль мастера матчится substring: `executorRole` в `specialization`
- `?debug=1` — verbose ответ с reason-кодами

### Клиенты (регистрация)
| Метод | Путь | Описание |
|-------|------|----------|
| POST | /api/clients/register | Создаёт pendingClient, возвращает verificationCode + botUsername |
| POST | /api/clients/login | Вход по phone + password |
| POST | /api/clients/verify-telegram-code | Привязывает Telegram к аккаунту |
| GET | /api/clients/verify-status?code= | Проверяет, привязан ли Telegram |

### Загрузка файлов
| Метод | Путь | Описание |
|-------|------|----------|
| POST | /api/upload | Загружает фото, сохраняет в public/uploads/services/ |

## Потоки данных

### Регистрация клиента (site-initiated)
```
1. Клиент: POST /api/clients/register → pendingClients + verificationCode
2. Сайт: показывает ссылку t.me/BOT?start=CODE
3. Клиент: /start CODE в боте → бот создаёт clients из pendingClients с telegramId
4. Сайт: GET /api/clients/verify-status?code=CODE → verified: true → localStorage
```

### Регистрация клиента (bot-initiated)
```
1. Клиент: открывает бота → не в clients → бот вставляет telegramVerificationCodes
2. Бот: ссылка SITE_URL/?tg_code=CODE
3. Сайт: видит ?tg_code → открывает форму регистрации
4. Клиент: POST /api/clients/register → pendingClients
5. Сайт: POST /api/clients/verify-telegram-code {code, phone} → clients с telegramId
6. Бот: "✅ Я зарегистрировался" → проверяет clients по telegramId
```

### Создание записи
```
1. Клиент: GET /api/available-slots?masterId=X&serviceId=Y&date=Z
2. Клиент: выбирает время → POST /api/appointments
3. Бот admin: уведомление администратора
4. За 1 час: cron/ручной запуск npm run reminders:send → Telegram сообщение клиенту
```

### Подтверждение рабочего слота мастером
```
1. Admin: POST /api/work-slots → workSlot + уведомление в masters-бот
2. Мастер: нажимает ✅ confirm_X → isConfirmed = true
3. Мастер: нажимает ❌ reject_X → удаляется
4. Мастер: запрашивает изменение → workSlotChangeRequests (type=time_change)
5. Admin: видит запросы в /admin → approves/rejects
```

## Дизайн-система

**Цвета:**
- Admin UI: `bg-[#0D0D11]`, violet/indigo градиенты
- Client site: `bg-[#09090D]`, бренд-красный `#B2223C → #e8556e`

**Шрифты:**
- `var(--font-playfair)` — заголовки
- `var(--font-montserrat)` — интерфейс/тело
- `var(--font-inter)` — код/числа

**Правила:**
- Модалки через `createPortal(modal, document.body)` с `z-[9999]`
- Даты: всегда `getFullYear/getMonth/getDate`, никогда `toISOString().slice(0,10)`
- DB import в ботах: `../db/index-sqlite` (не `../db`)
