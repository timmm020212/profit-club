# 🔍 **Отладка проблемы с услугами**

## 🐛 **Проблема:** В поле услуг показывается только "парикмахер" хотя в админке 3 услуги

## 🔍 **Что найдено:**

### **1. Услуги загружаются через API:**
```
GET /api/services
```

### **2. Компонент для отображения услуг:**
```
BookingServicesGrid.tsx - загружает и фильтрует услуги
```

### **3. Фильтрация в компоненте:**
```typescript
// Фильтрация по категории если указана
if (category) {
  filteredServices = filteredServices.filter(service => 
    service.category === category
  );
}

// Фильтрация по роли исполнителя если указана  
if (executorRole) {
  filteredServices = filteredServices.filter(service => 
    service.executorRole === executorRole
  );
}
```

## 🧪 **Что проверить:**

### **1. Проверьте API услуг:**
```
http://localhost:3000/api/services
```
Должен вернуть JSON со всеми услугами

### **2. Проверьте админку услуг:**
```
http://localhost:3000/admin/services
```
Посмотрите какие услуги там есть

### **3. Проверьте фильтры:**
Возможно компонент вызывается с фильтром:
- `category="парикмахер"`
- `executorRole="парикмахер"`

## 🔧 **Возможные проблемы:**

### **1. API возвращает только одну услугу**
```
Решение: Проверить API /api/services
```

### **2. Фильтрация срезает услуги**
```
Решение: Проверить параметры вызова BookingServicesGrid
```

### **3. Проблема с категориями/ролями**
```
Решение: Проверить что у услуг правильные category и executorRole
```

## 💡 **Что делать прямо сейчас:**

1. **Откройте** http://localhost:3000/api/services
2. **Посмотрите** сколько услуг возвращает API
3. **Откройте** http://localhost:3000/admin/services  
4. **Сравните** с тем что в API
5. **Проверьте** консоль на ошибки

```
API возвращает 3 услуги → BookingServicesGrid фильтрует по category/executorRole → Остается 1 услуга
```

**Проверьте API и фильтры - это определит где именно проблема!** 🔍
