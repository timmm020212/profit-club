# Обновленная система записи клиентов

## Что изменилось

### 1. Новая логика работы с расписанием

**Раньше:** Администратор создавал конкретные временные слоты (например, 10:00-11:00).

**Теперь:** Администратор создает рабочие дни для мастеров. Например, Маргарита Дереглазова работает 24 ноября с 8:00 до 18:00. Сайт автоматически рассчитывает доступные слоты для записи на основе:
- Рабочего дня мастера
- Длительности выбранной услуги
- Уже существующих записей

### 2. Обновленная схема базы данных

#### Таблица `services`
- Добавлено поле `duration` (длительность услуги в минутах)

#### Таблица `work_slots`
- Теперь хранит рабочие дни (весь день), а не отдельные слоты
- Удалено поле `isAvailable`
- `startTime` и `endTime` теперь обозначают начало и конец рабочего дня

#### Новая таблица `appointments`
- Хранит все записи клиентов
- Поля: masterId, serviceId, appointmentDate, startTime, endTime, clientName, clientPhone, status

### 3. Обновленный Telegram-бот

**Функционал:**
- ✅ Выбор мастера
- ✅ Выбор даты (с проверкой на месяц вперед)
- ✅ Указание рабочего дня (время начала и окончания)
- ✅ Предупреждение при перезаписи существующего рабочего дня
- ✅ Просмотр расписания (рабочих дней)

**Ограничения:**
- Нельзя выставлять расписание больше, чем на месяц вперед
- Нельзя указывать дату в прошлом

### 4. Новые API endpoints

#### `/api/available-slots`
**Параметры:** `masterId`, `serviceId`, `date`
**Описание:** Рассчитывает доступные временные слоты для записи с учетом:
- Рабочего дня мастера
- Длительности услуги
- Существующих записей

**Пример запроса:**
```
GET /api/available-slots?masterId=1&serviceId=1&date=2024-12-25
```

**Ответ:**
```json
[
  { "startTime": "08:00", "endTime": "09:00" },
  { "startTime": "08:30", "endTime": "09:30" },
  { "startTime": "09:00", "endTime": "10:00" }
]
```

#### `/api/appointments`
**GET** - получить список записей (с фильтрацией по masterId и date)
**POST** - создать новую запись

**Пример создания записи:**
```json
POST /api/appointments
{
  "masterId": 1,
  "serviceId": 1,
  "appointmentDate": "2024-12-25",
  "startTime": "10:00",
  "clientName": "Иван Иванов",
  "clientPhone": "+79001234567"
}
```

**Валидация при создании:**
- Проверка рабочего дня мастера
- Проверка, что время записи входит в рабочий день
- Проверка пересечений с существующими записями
- Автоматический расчет времени окончания на основе длительности услуги

### 5. Обновленные API endpoints

#### `/api/work-slots`
Теперь возвращает рабочие дни мастеров (не отдельные слоты).

#### `/api/masters`
Без изменений - возвращает список мастеров.

#### `/api/services`
Теперь включает поле `duration` (длительность в минутах).

## Как это работает

### Пример сценария:

1. **Администратор** через Telegram-бота устанавливает:
   - Мастер: Маргарита Дереглазова
   - Дата: 24.11.2024
   - Рабочее время: 08:00 - 18:00

2. **Клиент** на сайте выбирает:
   - Услугу: "Стрижка женская" (длительность 60 минут)
   - Мастера: Маргарита Дереглазова
   - Дата: 24.11.2024

3. **Сайт** запрашивает доступные слоты:
   ```
   GET /api/available-slots?masterId=10&serviceId=1&date=2024-11-24
   ```

4. **API** рассчитывает доступные слоты:
   - Берет рабочий день мастера (08:00-18:00)
   - Вычитает длительность услуги (60 минут)
   - Исключает занятые слоты
   - Возвращает доступные интервалы (08:00, 08:30, 09:00, ... 17:00)

5. **Клиент** выбирает время, например 10:00

6. **Сайт** создает запись:
   ```
   POST /api/appointments
   {
     "masterId": 10,
     "serviceId": 1,
     "appointmentDate": "2024-11-24",
     "startTime": "10:00",
     "clientName": "Иван Иванов"
   }
   ```

7. **API** автоматически:
   - Рассчитывает endTime (11:00)
   - Проверяет, что слот свободен
   - Создает запись

8. При следующем запросе доступных слотов слот 10:00-11:00 уже будет недоступен.

## Алгоритм расчета доступных слотов

1. Получаем рабочий день мастера на указанную дату
2. Получаем длительность выбранной услуги
3. Получаем все существующие записи мастера на эту дату
4. Генерируем слоты с интервалом 30 минут от начала до конца рабочего дня
5. Для каждого слота проверяем:
   - Достаточно ли времени до конца рабочего дня (startTime + duration <= endTime)
   - Не пересекается ли слот с существующими записями
6. Возвращаем список доступных слотов

## Что нужно сделать для запуска

1. **Применить обновленную схему БД:**
   ```bash
   npm run db:push
   ```

2. **Обновить услуги (добавить длительность):**
   ```bash
   npm run db:seed
   ```
   ⚠️ **Внимание:** Если услуги уже были добавлены ранее, нужно либо удалить их, либо обновить вручную через Drizzle Studio.

3. **Добавить мастеров (если еще не добавлены):**
   ```bash
   npm run db:seed-masters
   ```

4. **Запустить бота:**
   ```bash
   npm run bot:start
   ```

## Важные замечания

- **Интервал слотов:** По умолчанию доступные слоты генерируются с интервалом 30 минут. Можно изменить в `app/api/available-slots/route.ts` (переменная `slotInterval`)
- **Предотвращение двойных записей:** Система автоматически проверяет пересечения временных интервалов
- **Ограничение по месяцу:** Администратор не может выставить расписание больше, чем на месяц вперед
- **Автоматический расчет времени окончания:** При создании записи время окончания рассчитывается автоматически на основе длительности услуги
