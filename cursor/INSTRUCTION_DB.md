# Подробная инструкция по настройке PostgreSQL и Drizzle Studio

## Шаг 1: Установка PostgreSQL

### Вариант 1: Скачать с официального сайта (рекомендуется)

1. Перейдите на сайт: https://www.postgresql.org/download/windows/
2. Нажмите "Download the installer"
3. Выберите версию (рекомендуется последняя стабильная, например 16.x)
4. Скачайте установщик для Windows
5. Запустите установщик и следуйте инструкциям:
   - **Важно!** Запомните пароль, который вы установите для пользователя `postgres` (это будет суперпользователь)
   - Порт по умолчанию: `5432` (оставьте как есть)
   - Остальные настройки можно оставить по умолчанию

### Вариант 2: Установка через Chocolatey (если установлен)

```powershell
choco install postgresql
```

## Шаг 2: Проверка установки PostgreSQL

1. Откройте **pgAdmin 4** (устанавливается вместе с PostgreSQL)
   - Или найдите в меню Пуск "pgAdmin 4"
2. При первом запуске вас попросят установить мастер-пароль для pgAdmin
3. В левой панели найдите "Servers" → "PostgreSQL 16" (или ваша версия)
4. Нажмите на сервер и введите пароль, который вы установили при установке PostgreSQL

## Шаг 3: Создание базы данных

### Способ 1: Через pgAdmin (визуально)

1. В pgAdmin правой кнопкой мыши нажмите на "Databases"
2. Выберите "Create" → "Database..."
3. В поле "Database" введите: `profit_club`
4. Нажмите "Save"

### Способ 2: Через командную строку (psql)

1. Откройте командную строку (cmd) или PowerShell
2. Перейдите в папку PostgreSQL (обычно `C:\Program Files\PostgreSQL\16\bin`)
3. Или добавьте PostgreSQL в PATH
4. Выполните команду:

```bash
psql -U postgres
```

5. Введите пароль, который вы установили при установке
6. Выполните SQL команду:

```sql
CREATE DATABASE profit_club;
```

7. Выйдите из psql:

```sql
\q
```

## Шаг 4: Настройка проекта

### 4.1. Создайте файл `.env.local` в корне проекта

1. Откройте папку проекта в редакторе кода
2. Создайте файл `.env.local` (если его нет)
3. Добавьте в него следующую строку:

```
DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/profit_club
```

**Замените `ВАШ_ПАРОЛЬ` на пароль, который вы установили для пользователя postgres при установке PostgreSQL.**

Пример:
```
DATABASE_URL=postgresql://postgres:mypassword123@localhost:5432/profit_club
```

### 4.2. Структура строки подключения:

```
postgresql://[пользователь]:[пароль]@[хост]:[порт]/[база_данных]
```

- **пользователь**: обычно `postgres` (суперпользователь)
- **пароль**: пароль, который вы установили при установке PostgreSQL
- **хост**: `localhost` (если PostgreSQL на вашем компьютере)
- **порт**: `5432` (стандартный порт PostgreSQL)
- **база_данных**: `profit_club`

## Шаг 5: Применение схемы базы данных

Откройте терминал в папке проекта и выполните:

```bash
npm run db:push
```

Эта команда создаст таблицу `services` в вашей базе данных.

## Шаг 6: Заполнение базы данных начальными данными

Выполните команду:

```bash
npm run db:seed
```

Это добавит 8 услуг в базу данных.

## Шаг 7: Запуск Drizzle Studio

Выполните команду:

```bash
npm run db:studio
```

Через несколько секунд откроется браузер с адресом: **http://localhost:4983**

В Drizzle Studio вы сможете:
- Просматривать все услуги
- Редактировать существующие услуги
- Добавлять новые услуги
- Удалять услуги

## Шаг 8: Проверка работы сайта

1. Убедитесь, что dev-сервер запущен: `npm run dev`
2. Откройте http://localhost:3000
3. Услуги должны загружаться из базы данных

## Частые проблемы и решения

### Проблема: "Connection refused" или "Cannot connect to database"

**Решение:**
1. Убедитесь, что PostgreSQL запущен:
   - Откройте "Службы" (Services) в Windows
   - Найдите "postgresql-x64-16" (или ваша версия)
   - Убедитесь, что статус "Выполняется" (Running)
   - Если нет - запустите службу

2. Проверьте правильность пароля в `.env.local`

3. Проверьте, что база данных `profit_club` создана

### Проблема: "Database does not exist"

**Решение:**
Создайте базу данных (см. Шаг 3)

### Проблема: "Password authentication failed"

**Решение:**
1. Проверьте пароль в `.env.local`
2. Если забыли пароль, можно сбросить его через pgAdmin:
   - Правой кнопкой на пользователе `postgres` → Properties → Definition
   - Введите новый пароль

### Проблема: Drizzle Studio не открывается

**Решение:**
1. Убедитесь, что PostgreSQL запущен
2. Проверьте, что `.env.local` существует и содержит правильный `DATABASE_URL`
3. Попробуйте перезапустить: остановите (Ctrl+C) и запустите снова `npm run db:studio`

## Полезные команды

```bash

npm run db:push

# Заполнить БД данными
npm run db:seed

# Открыть Drizzle Studio
npm run db:studio

# Сгенерировать миграции (если нужно)
npm run db:generate
```

## Где найти данные для подключения

1. **Пользователь**: Обычно `postgres` (суперпользователь, создаётся при установке)
2. **Пароль**: Тот, который вы установили при установке PostgreSQL
3. **Хост**: `localhost` (если PostgreSQL на вашем компьютере)
4. **Порт**: `5432` (стандартный порт, можно проверить в pgAdmin)
5. **База данных**: `profit_club` (создаёте сами)
