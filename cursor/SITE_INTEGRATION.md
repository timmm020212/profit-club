## 🧪 Как проверить работу

### Шаг 1: Убедитесь, что в БД есть данные

1. **Добавьте рабочий день через бота:**
   - Откройте Telegram-бота
   - Добавьте рабочий день для любого мастера
   - Например: Маргарита Дереглазова, 25.12.2024, 08:00-18:00

2. **Проверьте в БД:**
   ```bash
   npm run db:studio
   ```
   - Откройте таблицу `work_slots` - должен быть рабочий день
   - Откройте таблицу `masters` - должны быть 14 мастеров
   - Откройте таблицу `services` - должны быть услуги с `duration`

### Шаг 2: Проверьте на сайте

1. **Откройте страницу записи:**
   ```
   http://localhost:3000/booking
   ```

2. **Выберите услугу:**
   - Нажмите "Записаться" на любой услуге
   - Должно открыться модальное окно

3. **Выберите дату:**
   - Выберите дату, на которую вы добавили рабочий день через бота
   - Нажмите "Выбрать время"

4. **Проверьте доступное время:**
   - Должны появиться реальные временные слоты
   - Слоты рассчитываются на основе:
     - Рабочего дня мастера (из бота)
     - Длительности услуги (из БД)
     - Существующих записей

5. **Выберите время:**
   - Выберите любой доступный слот
   - Нажмите "Выбрать мастера"

6. **Проверьте доступных мастеров:**
   - Должны появиться реальные мастера из БД
   - Показываются только те мастера, у которых есть свободный слот на это время

7. **Выберите мастера и создайте запись:**
   - Выберите мастера
   - Заполните имя и телефон
   - Нажмите "Подтвердить запись"
   - Запись должна создаться в БД

### Шаг 3: Проверьте, что слот занят

1. **Снова откройте запись на ту же услугу**
2. **Выберите ту же дату и время**
3. **Проверьте:**
   - Выбранный слот должен исчезнуть из доступных
   - Или мастер, на которого записались, не должен быть в списке доступных

## 🔍 Отладка

### Если не видно доступного времени:

1. **Проверьте, что рабочий день добавлен:**
   ```bash
   curl http://localhost:3000/api/work-slots
   ```

2. **Проверьте доступные слоты напрямую:**
   ```bash
   curl "http://localhost:3000/api/available-slots?masterId=10&serviceId=1&date=2024-12-25"
   ```
   (Замените masterId, serviceId и date на реальные значения)

3. **Проверьте консоль браузера:**
   - Откройте DevTools (F12)
   - Посмотрите вкладку Console
   - Должны быть запросы к API и возможные ошибки

### Если не видно мастеров:

1. **Проверьте API мастеров:**
   ```bash
   curl http://localhost:3000/api/masters
   ```

2. **Проверьте, что выбранное время есть в доступных слотах**

3. **Проверьте консоль браузера на ошибки**
