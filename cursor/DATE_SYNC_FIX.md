# Исправление синхронизации дат между БД и сайтом

## Проблема

Пользователь видит в консоли, что выбрана дата `2025-11-23`, но в БД рабочие дни на:
- Маргарита: `2024-11-24`
- Юлия: `2025-11-24` (было)

## Что было исправлено

1. ✅ **Исправлены даты в БД:**
   - Все даты 2025 года исправлены на 2024 год
   - Маргарита: `2024-11-24` ✅
   - Юлия: `2024-11-24` ✅ (исправлено)

2. ✅ **Исправлено форматирование даты в календаре:**
   - Убрано использование `toISOString()`, которое могло менять дату из-за часового пояса
   - Теперь дата форматируется напрямую из компонентов Date

3. ✅ **Улучшена логика календаря:**
   - Календарь правильно ограничивает даты месяцем вперед
   - Правильная проверка дат для следующих месяцев

## Текущее состояние БД

```bash
npm run db:studio
```

Или проверьте через скрипт:
```bash
npx tsx scripts/check-work-slots.ts
```

## Как проверить

1. **Откройте сайт:** `http://localhost:3000/booking`
2. **Выберите услугу** (например, "Стрижка мужская" - ID 2)
3. **Выберите дату:** 24 ноября 2024
4. **Проверьте доступное время:**
   - Должны быть слоты для Маргариты (ID 10)
   - Должны быть слоты для Юлии (ID 13)

## Если проблема сохраняется

1. **Проверьте консоль браузера (F12):**
   - Дата должна быть `2024-11-24`, а не `2025-11-23`
   - Если дата неправильная, обновите страницу (Ctrl+F5)

2. **Проверьте API напрямую:**
   ```bash
   curl "http://localhost:3000/api/work-slots"
   curl "http://localhost:3000/api/available-slots?masterId=10&serviceId=2&date=2024-11-24"
   ```

3. **Очистите кеш браузера** и перезагрузите страницу

## Важно

- Все рабочие дни должны быть в пределах месяца от текущей даты
- При добавлении через бота используйте формат `ДД.ММ.ГГГГ` (например, `24.11.2024`)
- Бот автоматически проверяет, что дата не больше месяца вперед
