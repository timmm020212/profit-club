## 🧪 Тестирование Telegram-бота

### Шаг 1: Найти бота в Telegram

1. Откройте Telegram
2. Найдите вашего бота (имя, которое вы указали при создании через @BotFather)
3. Или используйте токен бота для поиска

### Шаг 2: Протестировать бота

**Команда `/start`:**
- Отправьте боту `/start`
- Если ваш Telegram ID = `6140319824`, вы должны увидеть главное меню
- Если нет доступа - добавьте свой Telegram ID в таблицу `admins` в БД

**Проверка списка мастеров:**
- Нажмите кнопку "📋 Список мастеров"
- Должны увидеть всех 14 мастеров

**Добавление рабочего дня:**
1. Нажмите "➕ Добавить рабочее время"
2. Выберите мастера (например, "Маргарита Дереглазова - парикмахер-стилист (Premium Luxury Мастер)")
3. Введите дату в формате ДД.ММ.ГГГГ (например, `25.12.2024`)
4. Введите время начала рабочего дня (например, `08:00`)
5. Введите время окончания рабочего дня (например, `18:00`)
6. Бот должен подтвердить создание рабочего дня

**Проверка ограничений:**
- Попробуйте указать дату больше месяца вперед - должна появиться ошибка
- Попробуйте указать дату в прошлом - должна появиться ошибка

**Просмотр расписания:**
- Нажмите "📅 Просмотр расписания"
- Должны увидеть все рабочие дни, которые вы добавили

## 🌐 Тестирование API

### 1. Проверка списка мастеров

```bash
curl http://localhost:3000/api/masters
```

Или откройте в браузере:
```
http://localhost:3000/api/masters
```

Должны увидеть JSON с массивом мастеров.

### 2. Проверка списка услуг

```bash
curl http://localhost:3000/api/services
```

Должны увидеть услуги с полем `duration`.

### 3. Проверка рабочих дней

```bash
curl "http://localhost:3000/api/work-slots?masterId=10&date=2024-12-25"
```

(Замените `masterId=10` на реальный ID мастера из БД и `date` на дату, которую вы добавили через бота)

### 4. Расчет доступных слотов

```bash
curl "http://localhost:3000/api/available-slots?masterId=10&serviceId=1&date=2024-12-25"
```

**Что проверить:**
- Должен вернуть массив доступных слотов
- Слоты должны начинаться с начала рабочего дня
- Конец каждого слота = начало + длительность услуги

**Пример ответа:**
```json
[
  {"startTime": "08:00", "endTime": "09:00"},
  {"startTime": "08:30", "endTime": "09:30"},
  {"startTime": "09:00", "endTime": "10:00"},
  ...
]
```

### 5. Создание записи

```bash
curl -X POST http://localhost:3000/api/appointments \
  -H "Content-Type: application/json" \
  -d '{
    "masterId": 10,
    "serviceId": 1,
    "appointmentDate": "2024-12-25",
    "startTime": "10:00",
    "clientName": "Иван Иванов",
    "clientPhone": "+79001234567"
  }'
```

**Что проверить:**
- Запись должна создаться успешно
- В ответе должно быть `endTime` (рассчитанное автоматически)
- После создания записи, этот слот должен исчезнуть из доступных

### 6. Проверка, что слот занят

После создания записи, снова запросите доступные слоты:
```bash
curl "http://localhost:0/api/available-slots?masterId=10&serviceId=1&date=2024-12-25"
```

Слот `10:00-11:00` должен исчезнуть из списка.

### 7. Проверка списка записей

```bash
curl "http://localhost:3000/api/appointments?masterId=10&date=2024-12-25"
```

Должна быть видна созданная запись.

## 🐛 Частые проблемы

### Бот не отвечает
- Проверьте, что бот запущен: `npm run bot:start`
- Проверьте токен бота в коде
- Проверьте подключение к БД

### Ошибка доступа к боту
- Проверьте ваш Telegram ID
- Добавьте себя в таблицу `admins` в БД:
  ```sql
  INSERT INTO admins (telegram_id, full_name) VALUES ('ВАШ_ID', 'Ваше имя');
  ```

### API не работает
- Убедитесь, что Next.js сервер запущен: `npm run dev`
- Проверьте, что порт 3000 свободен (или используйте другой порт)

### Ошибки БД
- Проверьте подключение: `npm run db:studio`
- Проверьте `DATABASE_URL` в `.env.local`
